   29 --   Tools for parsing, pretty-printing, and compiling the Ninja build language.
   30 --
   31 --   This module re-exports some of the modules under the "Language.Ninja"
   32 --   namespace for convenience.
   33 --
   34 --   Take a look at "Language.Ninja.Tutorial" for a tutorial on how to use this
   35 --   library. If you just want to dive in, I recommend reading these modules
   36 --   in the following order:
   37 --
   38 --   1. Skim "Language.Ninja.Misc", "Language.Ninja.Mock",
   39 --      and "Language.Ninja.Errors" for familiarity with types and values
   40 --      that are used all over this library.
   41 --   2. Check out "Language.Ninja.Lexer" and then "Language.Ninja.Parser" to
   42 --      get an idea for what is possible with the lexer/parser API.
   43 --   3. Read "Language.Ninja.AST" to understand what the parsed AST looks like.
   44 --   4. Read "Language.Ninja.Compile" to see what the AST-to-IR compiler does.
   45 --   5. Read "Language.Ninja.IR" to understand what the compiled IR looks like.
   46 --   6. Read "Language.Ninja.Pretty" to see what the pretty-printer is capable
   47 --      of (spoiler: not very much).
   48 --
   49 --   If you are not already well-versed in the Ninja build language, it is also
   50 --   probably worth reading the Ninja
   51 --   <https://ninja-build.org/manual.html manual>.
   52 --
   53 --   When importing from this library, I recommend the following style:
   54 --
   55 --   > import qualified Language.Ninja     as Ninja
   56 --   > import qualified Language.Ninja.AST as Ninja.AST
   57 --   > import qualified Language.Ninja.IR  as Ninja.IR
   58 --
   59 --   Happy hacking!
   60 --
   61 --   @since 0.1.0
